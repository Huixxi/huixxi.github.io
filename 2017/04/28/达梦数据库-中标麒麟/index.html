达梦数据库的安装

首先说一下，较之人大金仓，个人感觉达梦数据库的技术文档更多也更全面，也很友好，同时也是国产为数不多的真正有自己技术的数据库产品。

1. 下载安装：

1.1 产品和文档下载：

可到达梦数据库官网下载Linux平台64位DM7试用版数据库，有一年的免费试用期，下载地址技术文档
主要文档有：DM Program.pdf、DM SQL.pdf、DM DBA.pdf

1.2 解压和安装：

下载到本地后，解压（tar -zxvf...），安装，可以根据解压文件夹下的DM_Install_zh.pdf安装文档—>“2.2Linux下DM服务器、客户端软件的安装”进行操作安装。

1.3 安装前准备：

安装达梦数据库的时候，会对比如/tmp的大小，以及系统资源限制等有比较严格的要求，这里给出具体的解决办法：

1.3.1.1 /tmp文件大小要求：

安装程序产生的临时文件，默认使用/tmp目录，要求必须要有大于600M的空间，可以在一开始安装中标麒麟的时候为/tmp文件夹手动分配1G+的空间；也可以设置环境变量修改其默认目录，修改方法如下：

DM_INSTALL_TMPDIR=“你要选择的临时路径”
Export DM_INSTALL_TMPDIR //变量添加到PATH中

1.3.1.2 修改系统资源限制：

首先用户使用dmdba系统用户进行登录

Uimit -a //查看用户资源限制
会发现open files(-n)一项不符合要求，（建议用户设置为65536以上或unlimited(无限制)。）如果你尝试
Uimit -Hn 65536直接修改其数值的话，会发现无法修改，需要修改两个配置文件,修改方法为：
【Root（超级用户） 下面，
修改内核文件：fs.file-max
# sysctl -w fs.file-max=500000 //增加open file limit to 500000
# vim /etc/sysctl.conf //打开相关配置文件
添加语句：fs.file-max=500000
# sysctl -p //执行
vim打开/etc/security/limits.conf文件，添加语句：
* hard nofile 65536
* soft nofile 65536
dmdba hard nofile 65536
dmdba soft nofile 65536
为防止limits.d/90_nporc_conf文件覆盖，我们需要往这个文件添加两条语句：
* hard nofile 65536
* soft nofile 65536
】

1.4 安装：

进入解压目录，dmdba身份运行： ./DMInstall.bin,按步骤安装即可

1.5 运行（在后台）：

安装文件夹dmdba/Dmdbms/tool/下有各种达梦的相关工具：
1．管理工具：Manager
2．数据迁移工具：Dts
3．控制台工具：Console
4．性能监控工具：Monitor
5．审计分析工具：Analyzer
6．ODBC 3.0驱动程序
7．JDBC 3.0驱动程序
8．C Language Tools：一组C语言开发的命令行工具
运行sh dbca.sh 注册、后台启动达梦数据库：

打开data/DAMENG/dm.ini注册文件，进行后台启动达梦数据库：

1.6 数据库迁移：

1.6.1.1

Navicat for Postgresql工具将PG中的表数据导出为sql文件

1.6.1.2

由于不同数据库之间的数据类型的不同，还需要对导出的sql文件进行必要修改，才能继续导入达梦数据库中，修改方法如下：

1.7 修改方法：

序列修改方法
CREATE SEQUENCE "MSFA（此处为达梦下的数据库的管理用户名）"."loot_id_seq" INCREMENT BY 1 START WITH 2 //达梦数据库从2开始 MAXVALUE 9223372036854775807 MINVALUE 1;
删除不支持的语句
DROP SEQUENCE IF EXISTS "MSF3"."api_keys_id_seq";
数据类型替换（相对应的不同数据类型需要做相应替换）
Int4 ------- int Bool ------ varchar(1) Inet ------- varchar(8188) //最大支持8188 Float8 ----- float
Insert into 语法修改，比如：
DM：INSERT INTO "hosts" PG： INSERT INTO "MSFA"."hosts" //注意区别

1.8 导入达梦数据库

dmdba 身份下，cd …/tool文件夹下：运行 ./manager
创建一个新的管理用户如：MSFA，然后打开我们修改后的sql文件进行导入，最终结果可在左侧—>模式—->MSFA—>表，进行查看。

中标麒麟操作系统安装

1. 下载安装：

1.1 下载：

到其官网，百度云盘免费下载中标麒麟桌面版本（iso文件，64位），大约1.85G。 云盘下载链接

1.2 安装过程：

可以选择物理机安装和虚拟机安装，但由于中标麒麟系统的兼容性问题，物理安装是容易失败的，表现为安装过程中画面停留在“红色麒麟图标” 状态或是“黑屏”状态，所以这里建议虚拟机安装，安装平台为“VMWare12+中标麒麟桌面版6.0”，打开后，点击“安装到硬盘”，一路默认安装即可。
【这里要提醒一下：
中标麒麟是基于Linux内核，准确的说是radhat 的 release5.6版本，从他的安装界面也可以看出，仍然保留着RedHat安装界面的风格；
内存和硬盘的容量分配：尽量从一开始分配的多一些，如内存3G+和硬盘40G+的样子，不然后面调整起来太过麻烦。】
1.3 安装后初步系统了解：
查看操作系统详细信息：
lsb_release -a
查看操作系统发行信息：
cat /etc/*release
查看操作系统位数：
getconf LONG_BIT
